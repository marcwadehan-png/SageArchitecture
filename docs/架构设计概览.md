# SageArchitecture 架构设计概览

## 一、系统定位

SageArchitecture 是面向超级智能体的**全协同架构系统**，核心理念：

> "贤者全就位，网状大协同，格子化记忆，全局智能体"

## 二、三大核心模块

### 2.1 神之架构 (God Architecture)

**核心职责**: 定义智能体的组织结构和协作关系

| 维度 | 规模 |
|------|------|
| 学派 | 42个 |
| 问题类型 | 165个 |
| 部门 | 11个 |
| 岗位 | 422个 |
| Claw | 763个 |
| 主线 | 7条 |

**七条主线**:
1. 皇家主线 - 最高决策
2. 文治主线 - 智慧中枢
3. 经济主线 - 数据支撑
4. 军政主线 - 执行调度
5. 标准主线 - 执行落地
6. 创新主线 - 增长引擎
7. 审核主线 - 质量保障

### 2.2 贤者工程 (Sage Wisdom)

**核心职责**: 积累和运用古今中外的智慧结晶

- 600+贤者蒸馏文档
- 16卷分类体系
- 覆盖: 儒家/道家/佛家/法家/纵横家/兵家/科学家/艺术家...

### 2.3 贤者Claw (Sage Claw)

**核心职责**: 提供763个可执行的AI能力单元

**Claw分类**:
- 按学派: 42学派的认知维度
- 按职能: 决策/分析/创作/执行/评估
- 按级别: 专员/领班/总监/尚书/侍郎

## 三、协同机制

### 3.1 四种协同类型

| 类型 | 说明 | 典型场景 |
|------|------|----------|
| 并行 (PARALLEL) | 多线同时处理独立子任务 | 文治+经济并行处理 |
| 串联 (SEQUENTIAL) | 流水线式前后衔接 | 皇家→文治→标准→审核 |
| 反馈 (FEEDBACK) | 执行结果反向影响调度 | 审核→皇家反馈 |
| 混合 (HYBRID) | 并行+串联自适应组合 | CRITICAL级全系统联动 |

### 3.2 调度流程

```
用户请求
    ↓
问题分类 (165类型)
    ↓
学派匹配 (42学派)
    ↓
Claw选取 (763选N)
    ↓
岗位分配 (422岗位)
    ↓
主线协同 (7主线)
    ↓
执行输出
```

## 四、技术架构

### 4.1 模块结构

```
sageclaw/
├── configs/              # 763个YAML配置
│   ├── 001_xxx.yaml
│   ├── 002_xxx.yaml
│   └── ...
├── src/
│   ├── claw.py            # ClawEngine入口
│   ├── _claw_engine.py    # 核心引擎
│   ├── _claw_router.py   # 路由调度
│   ├── _claw_architect.py # 架构管理
│   ├── context/          # 上下文管理
│   ├── dispatcher/       # 调度器
│   ├── reasoning/        # 推理引擎
│   └── semantic/         # 语义搜索
```

### 4.2 配置文件格式

```yaml
id: claw_001
name: 逻辑分析
department: 文治
school: 逻辑学派
capabilities:
  - 分析
  - 推理
court_position: 逻辑专员
priority: 10
```

## 五、使用示例

```python
from sageclaw import ClawEngine

# 初始化引擎
engine = ClawEngine(config_dir="configs")

# 执行任务
result = engine.execute(
    task="分析用户增长策略",
    context={"domain": "增长"}
)

print(result)
```

## 六、扩展指南

### 添加新学派

1. 在 `god-architecture/` 添加学派文档
2. 在 `sageclaw/configs/` 添加对应Claw配置
3. 更新 `claw.py` 的学派映射

### 添加新Claw

在 `sageclaw/configs/` 添加新YAML文件:

```yaml
id: claw_xxx
name: 新能力
department: 对应部门
school: 对应学派
capabilities: [...]
```
